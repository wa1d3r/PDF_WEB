# Web Interface

Микросервис **Web Interface** является единой точкой входа для пользователей системы. Сервис не имеет собственной базы данных, не хранит файлы на диске и полностью делегирует бизнес-логику внутренним микросервисам.

## Архитектура

Сервис построен на базе **FastAPI** и реализует концепцию безопасного серверного рендеринга.

## Конфигурация

Настройка сервиса осуществляется через переменные окружения или файл `.env`:

| Переменная | Описание | Значение по умолчанию |
| --- | --- | --- |
| `PROJECT_NAME` | Название сервиса | `Web Proxy` |
| `STORAGE_API_URL` | Базовый URL центрального хранилища | `http://storage` |
| `CTFD_API_URL` | Базовый URL игровой платформы CTFd | `http://ctfd` |
| `PDF_GEN_API_URL` | Базовый URL генератора отчетов | `http://pdf_gen` |

## API Справочник

### Системные эндпоинты

* **`GET /health`**: Проверка работоспособности сервиса.

### Клиентские эндпоинты (UI)

* **`GET /`**: Возвращает отрендеренную страницу входа (шаблон `web_index.html` из хранилища).
* **`POST /api/preview`**: Принимает токен, запрашивает статистику игрока и возвращает отрендеренный дашборд (шаблон `web_dashboard.html`).

### Экспорт данных

* **`POST /api/export`**: Принимает токен, валидирует его в CTFd и проксирует запрос на генерацию PDF. Возвращает бинарный поток (`application/pdf`).

## Безопасность

* Сервис не содержит папки `templates/` или статических файлов. Вся разметка поставляется доверенным узлом (Storage API) в формате Base64.

## Разработка и тестирование

### Установка зависимостей

```bash
pip install -r requirements.txt
```

### Запуск сервиса

```bash
uvicorn src.main:app --host 0.0.0.0 --port 8000
```

### Запуск тестов

Для проверки интеграции клиентов и логики рендеринга используется `pytest`.

```bash
pip install -r requirements-dev.txt
pytest tests/
```

## Развертывание в Docker

### Сборка образа

```bash
docker build -t ctf-web-interface .
```

### Запуск контейнера

Для корректной работы сервиса необходимо передать URL-адреса смежных микросервисов через переменные окружения.

```bash
docker run -d \
  --name web-interface \
  -p 8080:8000 \
  -e STORAGE_API_URL="http://storage:8000" \
  -e CTFD_API_URL="http://ctfd:8000" \
  -e PDF_GEN_API_URL="http://pdf_gen:8000" \
  ctf-web-interface
```
