# Signer API

Микросервис **Signer API** отвечает за наложение визуальных штампов и криптографических электронных цифровых подписей (ЭЦП стандарта PKCS#12) на PDF-документы. Сервис динамически загружает необходимые ресурсы (текст, печать, закрытый ключ) из внешних источников (Storage API) и обеспечивает проверку безопасности входящих URL-адресов для защиты от SSRF-атак (кроме редиректа).

## Архитектура

Сервис построен на базе **FastAPI** и использует асинхронную обработку сетевых запросов и криптографических операций.

### Ключевые компоненты:

* **PDF Crypto Signer**: Ядро криптографии на базе `pyHanko`. Осуществляет парсинг PKCS#12 контейнеров, инкрементальную запись PDF-файлов и наложение визуальных штампов. Кэширует инициализированных подписантов в оперативной памяти для исключения повторного ресурсоемкого парсинга сертификатов.
* **Signature Asset Fetcher**: Безопасный асинхронный загрузчик ресурсов (`aiohttp`), скачивающий публичные шаблоны и приватный ключ подписи. Автоматически прикрепляет системный токен `X-Service-Token` для доступа к закрытым эндпоинтам.
* **WAF (Web Application Firewall)**: Встроенный механизм защиты уровня приложения, валидирующий и блокирующий вредоносные или подозрительные URL-адреса, переданные пользователем.

## Конфигурация

Настройка сервиса осуществляется через переменные окружения или файл `.env`:

| Переменная | Описание | Значение по умолчанию |
| --- | --- | --- |
| `PROJECT_NAME` | Название сервиса | `Signature API` |
| `SIGNER_TOKEN` | Токен для доступа к внутренним секретам Storage API | `token` |
| `MAX_DOCUMENT_SIZE` | Максимальный размер загружаемого ассета (в байтах) | `51200` |
| `PRIVATE_KEY_URL` | Внутренний URL для скачивания PKCS#12 контейнера | `http://172.17.0.5` |
| `PRIVATE_KEY_PASSWORD` | Пароль для расшифровки PKCS#12 контейнера | `password` |

## API Справочник

### Системные эндпоинты

* **GET `/health`**: Проверка работоспособности сервиса.

### Криптография

* **POST `/api/v1/sign`**: Подписать PDF документ.
* **Тело запроса (JSON)**:
* `document_base64` (str): Исходный PDF-документ в формате Base64.
* `text_url` (str): Ссылка на текстовый файл с содержимым штампа.
* `img_url` (str): Ссылка на изображение печати.


* **Успешный ответ (200 OK)**:
* `signed_document_base64` (str): Подписанный PDF-документ в формате Base64.


* **Возможные ошибки**: `403 Forbidden` (блокировка WAF), `422 Unprocessable Entity` (ошибки валидации или криптографии), `502 Bad Gateway` (ошибки сети).



## Безопасность (WAF & Limits)

Сервис реализует паттерн глубокой защиты для предотвращения SSRF и истощения ресурсов:

1. **Ограничение протоколов**: Разрешены исключительно схемы `http` и `https`.
2. **Блокировка Internal URL**: WAF отклоняет любые пользовательские ссылки, содержащие в пути директорию `/internal/` (независимо от регистра).
3. **Защита от обфускации**: URL, закодированные более 3-х раз, блокируются.
4. **Символьные ограничения**: Запрещены non-ASCII символы и матричные параметры (`;`).
5. **Контроль размера (Payload Limits)**: проверка `MAX_DOCUMENT_SIZE`.

*Используется конфигурация, уязвимая к SSRF через редиректы*

## Разработка и тестирование

### Установка зависимостей
```
pip install -r requirements.txt
```

### Запуск сервиса
```
uvicorn src.main:app --host 0.0.0.0 --port 8000
```

### Запуск тестов
```
pip install -r requirements-dev.txt
pytest tests/
```


## Развертывание в Docker

### Сборка образа

```bash
docker build -t ctf-signer-api .
```

### Запуск контейнера

Для корректной работы сервиса необходимо передать конфигурацию окружения для связи с хранилищем:

```bash
docker run -d \
  --name signer-api \
  -p 8001:8000 \
  -e SIGNER_TOKEN="your_storage_domain.your_hmac_signature" \
  -e PRIVATE_KEY_URL="http://storage:8000/internal/secrets/pkcs12" \
  -e PRIVATE_KEY_PASSWORD="cert_password" \
  ctf-signer-api
```

### Проверка статуса

```bash
curl http://localhost:8001/health
```
