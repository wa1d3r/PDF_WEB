# Storage API

Микросервис **Storage API** является централизованным хранилищем для всей статической разметки, шаблонов документов и секретных данных в рамках архитектуры проекта. Сервис обеспечивает разделение доступа к публичным ассетам и внутренним секретам с использованием криптографической проверки источника запроса.

## Архитектура

Сервис построен на базе **FastAPI** и использует локальное файловое хранилище в формате JSON для обеспечения высокой скорости работы и простоты развертывания.

### Ключевые компоненты:

* **JSON Storage**: Постоянное хранилище данных, загружаемое в оперативную память при старте сервиса.
* **Network Access Control (NAC)**: Система защиты внутренних эндпоинтов, проверяющая HMAC-SHA256 подпись токена и соответствие IP-адреса отправителя DNS-записи разрешенного домена.
* **Public API**: Доступ к шаблонам оформления и общедоступным ресурсам.
* **Internal API**: Защищенный доступ к чувствительным данным.



## Структура данных хранилища

Данные хранятся в файле, путь к которому определяется переменной `STORAGE_FILE_PATH` (по умолчанию `data.json`). Файл должен иметь следующую структуру:

```json
{
  "public": {
    "имя_объекта": "base64_кодированные_данные"
  },
  "internal": {
    "имя_объекта": "base64_кодированные_данные"
  }
}
```

## Конфигурация

Настройка сервиса осуществляется через переменные окружения или файл `.env`:

| Переменная | Описание | Значение по умолчанию |
| --- | --- | --- |
| `PROJECT_NAME` | Название сервиса | `Storage API` |
| `STORAGE_FILE_PATH` | Путь к JSON-файлу БД | `data.json` |
| `MASTER_SECRET` | Ключ для валидации токенов | `secret-key` |
| `ALLOWED_DOMAINS` | Список доверенных доменов (через запятую) | `[]` |

## API Справочник

### Системные эндпоинты

* **`GET /health`**: Проверка работоспособности сервиса.



### Публичные ассеты

* **`GET /public/assets/{asset_name}`**: Возвращает Base64 содержимое объекта из секции `public`.



### Приватные секреты

* **`GET /internal/secrets/{secret_key}`**: Возвращает Base64 содержимое из секции `internal`.

    * **Требование**: Заголовок `X-Service-Token`.
    * **Формат токена**: `{allowed_domain}.{hmac_signature}`.





## Безопасность (NAC)

Доступ к внутренним ресурсам разрешен только микросервисам, чьи домены указаны в `ALLOWED_DOMAINS`. Валидация включает:

1.  **Проверку подписи**: HMAC-SHA256 от имени домена с использованием `MASTER_SECRET`.

2. **DNS-сверку**: IP-адрес отправителя запроса должен совпадать с результатом `socket.gethostbyname({domain})`.



## Разработка и тестирование

### Установка зависимостей

```bash
pip install -r requirements.txt
```

### Запуск сервиса

```bash
uvicorn src.main:app --host 0.0.0.0 --port 8000
```

### Запуск тестов

Для проверки логики NAC и API используется `pytest`.

```bash
pip install -r requirements-dev.txt
pytest tests/
```
